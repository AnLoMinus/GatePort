## 🏷 Semantic Versioning

הפרויקט עוקב אחרי [Semantic Versioning](https://semver.org/lang/he/):

- **MAJOR** (X.0.0) – שינויים לא תואמים
- **MINOR** (0.X.0) – תכונות חדשות תואמות לאחור
- **PATCH** (0.0.X) – תיקוני באגים תואמים לאחור

## 📝 דוגמה לתהליך

### גרסה 0.1.0

```bash
# 1. עדכון VERSION
echo "0.1.0" > VERSION

# 2. עדכון CHANGELOG.md
# הוספת כל השינויים תחת [0.1.0]

# 3. Commit
git add VERSION CHANGELOG.md
git commit -m "chore: prepare release 0.1.0"

# 4. יצירת Tag
git tag -a v0.1.0 -m "Release version 0.1.0"

# 5. Push
git push origin main
git push origin v0.1.0
```
